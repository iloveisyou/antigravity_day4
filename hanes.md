# 페에지 제작 규칙 및 개발 가이드

## 규칙1: 이전에 한번도 실행이 안되었거나 수정이 된 내용은 실행해주고 나머지는 스킵해줘
- prompt.md: 입력하는 모든 프롬프트는 날짜별로 정리하고 프롬프트와 결과(적용내용) 등으로 정리해줘
- Implementation Plan 파일들을 implementation폴더에 plan+날짜+시간.md 파일로 저장 (예: plan202606301630.md)
- walkthrough.md 파일들을 walkthrough폴더에 날짜+시간.md 파일로 저장 (예: 202606301630.md)
- 내가 위에 적지 않았어도 나중에 문제가 생겼거나 내용을 파악하기에 좋다고 생각되는 문서들을 정리해서 저장해줘. 모든 문서는 마크다운 파일로 저장해
- github에 antigravity_day4 프로젝트랑 연결해줘, 없다면 새로운 프로젝트르 만들어서 연결해줘
- vercel에 antigravity-day4 프로젝트에 배포해주고, 없으면 새로운 프로젝트로 만들어서 베포해줘
- 모든 코드는 보안에 주의하면서 작성해줘. API 키 등 민감한 정보는 코드에 직접 포함하지 말고 반드시 환경 변수를 사용해줘. .env 파일을 추가하고 .gitignore 에 추가해줘. .env.example 파일도 추가해서 환경변수의 구성을 알려줘.
- 모든 단계 진행 후, 구현 계획과 실행 결과를 prompt.md에 기록해줘.
- 깃에 커밋 푸시하면 배포도 자동으로 진행해줘


## step1
- 'AI 감정 일기' 앱을 만들 거야.
1. '오늘의 하루는 어땟나요?'라는 제목을 보여줘.
2. 사용자가 여러 줄의 텍스트를 입력할 수 있는 넓은 입력창(textarea) 을 만들어줘.
3. 입력창 아래에 '음성으로 입력하기' 버튼과 '분석 요청하기' 버튼을 만들어줘.
4. 가장 아래에는 'AI의 답변' 이라는 소제목과 함께, AI의 답변이 표시될 회색 배경의 박수를 만들어줘. 처음에는 '여기에 AI의 답변이 표시됩니다.' 라고 쓰여있게 해줘.

## step2
- 이제 음성 인식 기능을 추가해줘.
- 사용자가 '음성으로 입력하기' 버튼을 클릭하면, 브라우저의 내장 음성 인식 기능을 활성화해줘. (Web Speech API를 사용하면 돼)
- 사용자가 말을 하면, 그 음성을 텍스트로 변환해서 넓은 입력창에 자동으로 채워 넣어줘.

## step3
- 이제 Google Gemini API를 사용할건데 내가 발급받은 API키를 안전하게 설정하고 싶어.
- 환경 변수를 사용해서 'GEMINI_API_KEY'라는 이름으로 이 키를 저장하고, 코드에서 불러와서 사용할 수 있게 해줘

## step4
- '분석 요청하기' 버튼을 클릭했을 때의 동작을 만들자.
1. 사용자가 입력창에 작성한 일기 내용을 가져와
2. 설정한 Gemini API 키를 사용해서, Gemini API를 호출해줘.
3. API를 호출할때. 다음과 같음 프롬프트를 전달해야 해:
"너는 심리 상담가야. 사용자가 작성한 일기 내용을 읽고, 사용자의 감정을 한단어(예: 기쁨, 슬픔, 분노, 불안, 평온)로 요약해줘. 그리고 그 감정에 공감해주는, 따뜻한 응원의 메시지를 2~3문장으로 작성해줘. 답변 형식은 반드시 '감정: [요약된 감정]\n\n[응원 메시지]' 와 같은 줄바꿈을 포함해서 보내줘"
4. API로부터 받은 답변을 AI의 답변이 표시되는 곳에 표시해줘.
5. .env의 GEMINI_API_KEY 환경변수 입력했어. 해당 환경변수를 로직에 적용해줘. 최신 flash 모델을 사용할거야.

## step5
- 이제 일기내용과 AI답변을 브라우저의 로컬 스토리지에 저장해서, 새로고침해도 내용이 유지되게 해줘.
1. 사용자가 '분석 요청하기'를 클릭하면, 일기내용과 AI답변을 로컬 스토리지에 저장해줘.
2. 페이지가 처음 로드될 때, 로컬 스토리지에 저장된 내용이 있는지 확인하고, 만약 있다면 그내용을 입력창과 AI 답변 박스에 각각 채워 넣어줘.
