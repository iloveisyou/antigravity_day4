# 페에지 제작 규칙 및 개발 가이드

## 규칙1: 이전에 한번도 실행이 안되었거나 수정이 된 내용은 실행해주고 나머지는 스킵해줘

- github에 antigravity_day4 프로젝트랑 연결해줘, 없다면 새로운 프로젝트르 만들어서 연결해줘
- vercel에 antigravity-day4 프로젝트에 배포해주고, 없으면 새로운 프로젝트로 만들어서 베포해줘
- 모든 코드는 보안에 주의하면서 작성해줘. API 키 등 민감한 정보는 코드에 직접 포함하지 말고 반드시 환경 변수를 사용해줘. .env 파일을 추가하고 .gitignore 에 추가해줘. .env.example 파일도 추가해서 환경변수의 구성을 알려줘.
- 모든 단계 진행 후, 구현 계획과 실행 결과를 prompt.md에 기록해줘.
- 깃에 커밋 푸시하면 배포도 자동으로 진행해줘

## 규칙2: AI과 진행하는 모든 과정은 사용자가 한눈에 볼수 있어야 한다.
- AI가 진행하면서 나오는 모든 임시 파일들은 폴더별 또는 파일별로 정리해야한다.
- 내용이 많은 문서는 폴더로 날짜+시간 파일명으로 저장해야 한다
- 내용이 적은 문서는 폴더에 파일 한개로 날짜+시간 파일명으로 저장해야한다. (단, 파일 한개로 저장하다 너무 많이 쌓이면 새로운 파일로 저장하도록 유도)
- 예시1(내용이 많은 경우): Implementation Plan 파일을 생성하는 경우 implementationPlan 폴더생성 후 202607071717.md 파일로 저장후, 새로 만들때마다 새로운 파일로 저장
- 예시2(내용이 적은 경우): Task 파일을 생성하는 경우 task 폴더 생성후 202607070917.md 파일 저장 후, 해당파일 안에서 날짜별로 정리, 그 안에서도 내용이 너무 많아지면 새로운 파일로 저장
- 예시3(내용이 많은 경우): Walkthorough 파일을 생성하는 경우 walkthorough 폴더 생성후, 202607070920.md 로 저장, 새로 만들때마다 새로운 파일로 계속 저장
- prompt.md: 입력하는 모든 프롬프트는 날짜별로 정리하고 프롬프트와 결과(적용내용) 등으로 정리해줘
- 내가 위에 적지 않았어도 나중에 문제가 생겼거나 내용을 파악하기에 좋다고 생각되는 문서들을 정리해서 저장해줘. 모든 문서는 마크다운 파일로 저장해 주고, 나에게 말해줘



## step1
- 'AI 감정 일기' 앱을 만들 거야.
1. '오늘의 하루는 어땟나요?'라는 제목을 보여줘.
2. 사용자가 여러 줄의 텍스트를 입력할 수 있는 넓은 입력창(textarea) 을 만들어줘.
3. 입력창 아래에 '음성으로 입력하기' 버튼과 '분석 요청하기' 버튼을 만들어줘.
4. 가장 아래에는 'AI의 답변' 이라는 소제목과 함께, AI의 답변이 표시될 회색 배경의 박수를 만들어줘. 처음에는 '여기에 AI의 답변이 표시됩니다.' 라고 쓰여있게 해줘.

## step2
- 이제 음성 인식 기능을 추가해줘.
- 사용자가 '음성으로 입력하기' 버튼을 클릭하면, 브라우저의 내장 음성 인식 기능을 활성화해줘. (Web Speech API를 사용하면 돼)
- 사용자가 말을 하면, 그 음성을 텍스트로 변환해서 넓은 입력창에 자동으로 채워 넣어줘.

## step3
- 이제 Google Gemini API를 사용할건데 내가 발급받은 API키를 안전하게 설정하고 싶어.
- 환경 변수를 사용해서 'GEMINI_API_KEY'라는 이름으로 이 키를 저장하고, 코드에서 불러와서 사용할 수 있게 해줘

## step4
- '분석 요청하기' 버튼을 클릭했을 때의 동작을 만들자.
1. 사용자가 입력창에 작성한 일기 내용을 가져와
2. 설정한 Gemini API 키를 사용해서, Gemini API를 호출해줘.
3. API를 호출할때. 다음과 같음 프롬프트를 전달해야 해:
"너는 심리 상담가야. 사용자가 작성한 일기 내용을 읽고, 사용자의 감정을 한단어(예: 기쁨, 슬픔, 분노, 불안, 평온)로 요약해줘. 그리고 그 감정에 공감해주는, 따뜻한 응원의 메시지를 2~3문장으로 작성해줘. 답변 형식은 반드시 '감정: [요약된 감정]\n\n[응원 메시지]' 와 같은 줄바꿈을 포함해서 보내줘"
4. API로부터 받은 답변을 AI의 답변이 표시되는 곳에 표시해줘.
5. .env의 GEMINI_API_KEY 환경변수 입력했어. 해당 환경변수를 로직에 적용해줘. 최신 flash 모델을 사용할거야.

## step5
- 이제 일기내용과 AI답변을 브라우저의 로컬 스토리지에 저장해서, 새로고침해도 내용이 유지되게 해줘.
1. 사용자가 '분석 요청하기'를 클릭하면, 일기내용과 AI답변을 로컬 스토리지에 저장해줘.
2. 페이지가 처음 로드될 때, 로컬 스토리지에 저장된 내용이 있는지 확인하고, 만약 있다면 그내용을 입력창과 AI 답변 박스에 각각 채워 넣어줘.

## 해당기점으로: 기존꺼 업데이트 서비리스 환경으로 업그레이드 할려 했지만, 이미 AI가 이 책을 쓰기 전과 다르게 업그레이드 되어서 인지 이미 서버리스 환경까지 진행해서 만들어줌

## step6. API 라우터(주문서 접수처) 만들기
- vercel 서버리스 함수르 만들고 싶어.
- 프로젝트에 'api'라는 폴더를 만들고, 그안에 'analyze.js' 서버리스 함수 안으로 옮겨줘.
- 이 파일은 앞으로 우리 서비스의 백엔드 API 역할을 할거야. 사용자의 일기 내용을 입력받아서, Gemini API를 호출하고, 그 겨로가를다시 들려주는 일을 해야해.
- 지금은 예시 파일만 만들어줘.

## step7. 핵심 로직, 백엔드로 이사시키기
- 기존에 만들었던 Gemini API 호출 기능을 'api/analyze.js' 서버리스 함수 안으로 옮겨줘.
- 이제부터 Gemini API 호출은 프론트엔드가 아니라 이 백엔드 함수에서만 일어나야 해.
- 그리고 가장 중요한 것 'GEMINI_API_KEY' 환경 변수를 프론트엔드가 접근할 수 없고, 오직 Vercel의 서버리스 환경에서만 사용할 수 있도록설정을 변경해줘.
- 코드베이스를 분석해서 Vercel 서버리스 함수에 최적화된 형태로 구현해줘.

## step8. 프롡트엔드에서 우리 API 호출하기
- 코드에 반영되어 있는지 먼저 확인하고 작업해
- 이제 프론트엔드의 '분석 요청하기' 버튼의 동작을 수정하자.
- 더이상 Gemini API 직접 호출하지 않고, 대신 우리가 만든 '/api/analyze' API에 POST방식으로 요청을 보내게 해줘.
- 요청을 보낼때는 사용자가 입력할 일기 내용을 함께 담아서 보내야 해.
- 그리고 우리 API로부터 응답이 오면, 그 내용을 AI 답변 박스에 표시해줘.

## step9. 서버리스 함수에서 데이터 저장하고 불러오기
- 'api/analyze.js' 서버리스 함수에 기능 추가해줘
- Gemini API로 부터 성공적으로 답변을 받은 후에, Vercel 마켓 플레이에서 연결한 Serverless Redis에 저장해줘. 환경변수는 REDIS_URL 이야
- 사용자의 원본 일기 내용과 AI의 답변을 하나의 묶음으로 해서, 현재 시간을 기준으로 고유한 ID를 만들어 저장해줘
- 예를들어 'aiary-20260708161300'와 같은 키(key)에 일기 데이터(value)를 저장하는 방식이야
- vercel dev을 실행하여 프로젝트의 환경변수를 가져와서 적용해. 관련 최신 정보는 웹검색을 통해 탐색한 다음 적용해

## step10. 저장된 히스토리 불러오기
- (현재는 서버에 저장만 하고, 서버데이터를 불러오고 있지 않음, 아직 브라우저의 로컬스토리지를 사용 함)
- 이제 프론트엔드에 저장된 일기 히스토리를 보여주는 기능을 만들자.
1. AI답변 박스 아래에 '나의 일기 히스토리' 라는 제목의 섹션을 만들어줘.
2. 페이지가 로드될 때, Vercel마켓플레이스 Serverless Redis에 저장된 모든ㅇ ㅣㄹ기 데이터를 가져오는 새로운 API 함수를 만들어줘. (예: /api/history)
3. 가져온 데이터를 최신순으로 정렬해서, 각 항목을 카드 형태로 보여줘, 카드에는 일기 내용과 AI의 답변이 모두 포함되어야해.
과련 최신정보는 웹검색을 통해탐색한 다음 적용해. 그런 다음 변경 사항 커밋 남기고 푸시해.
