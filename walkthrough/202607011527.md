## 구현 사항

### 1. 보안용 Vercel Serverless Function 도입
- [api/analyze.js](file:///Users/iloveisyou/Desktop/Project/AI/antigravity/day4/api/analyze.js) 파일을 새로 만들어 클라이언트에서 직접 구글 API를 호출하지 않고 백엔드 단에서 프록시 호출을 수행하도록 변경했습니다.
- `process.env.GEMINI_API_KEY` 환경 변수를 불러와 구글 `gemini-2.5-flash` 모델을 사용하여 감정 분석과 매끄러운 답변/위로의 글을 추출하도록 설정했습니다.
- Gemini API가 안정적인 JSON 응답을 보낼 수 있게 `responseMimeType: "application/json"` 형식을 사용하도록 API 요청 body를 설계했습니다.

## 검증 방법
1. 로컬 환경 변수 설정: 생성된 [.env](file:///Users/iloveisyou/Desktop/Project/AI/antigravity/day4/.env) 파일의 `YOUR_GEMINI_API_KEY_HERE` 값을 실제 발급받은 Google Gemini API 키로 교체합니다.
2. 로컬 실행: 터미널에서 `npx vercel dev` 명령어를 입력해 Vercel 로컬 서버를 실행하면 서버리스 함수와 프론트엔드가 함께 로컬 포트(기본 3000번)에서 연동되어 테스트가 가능해집니다.
3. Vercel 배포 시: Vercel 대시보드 -> Settings -> Environment Variables 메뉴에서 `GEMINI_API_KEY` 환경 변수를 등록해준 뒤 새로 배포(Vercel Deploy)하면 실제 상용 환경에서도 완벽히 보안이 지켜진 상태로 동작합니다.
