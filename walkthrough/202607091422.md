## 수행 결과
- **의존성 설치 및 환경변수 동기화**:
  - `package.json` 파일에 백엔드용 `@supabase/supabase-js` 패키지 버전을 추가하고 설치를 완료했습니다.
  - Vercel CLI로 가져온 Supabase 주소 및 Key 정보(`NEXT_PUBLIC_SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY` 등)를 로컬 개발 편의를 위해 `.env` 파일에 기록하고, 공통 예시를 위해 `.env.example`에도 명세를 추가했습니다.
- **백엔드 클라이언트 구현 (`api/_lib/supabase.js`)**:
  - Vercel Node.js 서버리스 함수 내부에서 사용하도록 `api/_lib/supabase.js` 모듈을 신규 생성했습니다.
  - 서버 측 환경변수(`SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`)를 읽어와 DB 우회 접근 권한을 갖는 백엔드 전용 `supabase` 클라이언트를 인스턴스화했습니다.
- **프론트엔드 연동 (`index.html`, `supabase.js`)**:
  - `index.html` 하단에 Supabase SDK 공식 CDN 스크립트 태그를 탑재했습니다.
  - 브라우저 전역 객체인 `window.supabaseClient`를 초기화하는 `supabase.js` 파일을 신설하여, 클라이언트 사이드에서 언제든지 사용자 인증 및 Supabase API 호출이 가능하도록 설계했습니다.

## 검증 결과
- **로컬 검증 완료**: `npx vercel dev` 구동 하에 웹페이지에 접근 시 브라우저에서 `supabaseClient` 객체가 정상 등록되고 로딩 에러 없이 성공적으로 Supabase CDN 및 로컬 설정 파일이 배포되어 연동됨을 확인했습니다.
